# External Integrations

**Analysis Date:** 2026-04-06 (updated after greenfield module refactor)

## APIs & External Services

**Voice & Telephony:**
- Twilio - Voice API for PSTN and WebRTC calls
  - SDK: `twilio` 9.0.0+
  - Auth: `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`
  - Configuration: `TWILIO_PHONE_NUMBER` (E.164 format)
  - Key files: `shuo/phone.py` (TwilioPhone, dial_out), `shuo/web.py` (TwiML endpoints)
  - Features: Outbound calls, WebSocket media streams, DTMF tones

**Speech-to-Text:**
- Deepgram Flux API - Real-time streaming STT + turn detection
  - SDK: `deepgram-sdk` 3.0.0+
  - Auth: `DEEPGRAM_API_KEY`
  - Key files: `shuo/speech.py` (Transcriber, TranscriberPool)
  - Connection: Persistent WebSocket per call
  - Audio format: μ-law 8kHz (direct from Twilio)
  - Turn detection: Automatic UserSpeakingEvent/UserSpokeEvent callbacks

**Large Language Model:**
- Groq API (OpenAI-compatible) - LLM inference
  - Framework: pydantic-ai (`pydantic-ai-slim[groq]`)
  - Auth: `GROQ_API_KEY`
  - Default model: `LLM_MODEL=llama-3.3-70b-versatile` (configurable)
  - Key files: `shuo/language.py` (LanguageModel)
  - Features: Streaming token response, conversation history, typed tool calls
  - Tools: `press_dtmf()`, `go_on_hold()`, `signal_hangup()`

**Text-to-Speech - Cloud:**
- ElevenLabs - Premium cloud TTS
  - Auth: `ELEVENLABS_API_KEY`
  - Voice ID: `ELEVENLABS_VOICE_ID` (default: Rachel `21m00Tcm4TlvDq8ikWAM`)
  - Model: `ELEVENLABS_MODEL=eleven_flash_v2_5` (configurable)
  - Key files: `shuo/voice_elevenlabs.py`
  - Protocol: WebSocket streaming to Twilio-compatible μ-law 8kHz

**Text-to-Speech - Local (Optional):**
- Kokoro-82M - Local TTS (zero API cost)
  - Deployment: Docker container
  - Key files: `shuo/voice_kokoro.py`
  - Environment: `TTS_PROVIDER=kokoro`

**Text-to-Speech - Self-hosted (Optional):**
- Fish Audio S2
  - Key files: `shuo/voice_fish.py`
  - Environment: Fish Audio server URL in configuration
  - Use case: Full privacy, self-hosted alternative to ElevenLabs

## Data Storage

**Databases:**
- None — voice agent is stateless
- Call state stored in-memory during active calls

**File Storage:**
- Call traces: `/tmp/shuo/{stream_sid}.json` for latency analysis
- Auto-rotated at startup via `cleanup_traces()` in `shuo/tracer.py`
  - Configurable: `TRACE_MAX_FILES` (default: 100), `TRACE_MAX_AGE_HOURS` (default: 24)

**In-Memory Event Bus:**
- `monitor/bus.py` - Call-scoped event bus for live transcription/phase updates
- `monitor/registry.py` - In-memory call registry (active calls)
- `asyncio.Queue` for event backpressure

## Authentication & Identity

**Twilio Webhook Validation:**
- `verify_twilio_signature` FastAPI dependency in `shuo/web.py`
- Validates `X-Twilio-Signature` header on all `/twiml*` POST routes
- Skipped when `TWILIO_AUTH_TOKEN` is unset (development convenience)

**Dashboard Authentication:**
- `verify_api_key` FastAPI dependency in `monitor/server.py`
- Checks `X-API-Key` header against `DASHBOARD_API_KEY` env var
- Skipped when env var is unset

**Rate Limiting:**
- In-process `_RateLimiter` in `monitor/server.py`
- Sliding window; configurable via `CALL_RATE_LIMIT` env var
- Applied to `/dashboard/call` endpoint

**Browser Softphone Token:**
- Generated by `shuo/web.py:get_token()` endpoint
- Signed with Twilio API Key + Secret (`TWILIO_API_KEY`, `TWILIO_API_SECRET`)

## Monitoring & Observability

**Logs:**
- Custom colored terminal logging via `shuo/log.py`
- `Logger` class for lifecycle events; `get_logger()` for module loggers
- No external log aggregation (logs to stdout)

**Performance Tracing:**
- Per-call latency tracing: `shuo/tracer.py`
- Traces saved to `/tmp/shuo/{stream_sid}.json`
- Spans: LLM first token, TTS first audio, playback complete
- Accessible via `GET /trace/latest` endpoint

## CI/CD & Deployment

**Hosting:**
- Railway (primary) - Python app with `Procfile` at repo root
- Any Python 3.9+ ASGI host (Heroku, Fly.io, AWS, etc.)

**Deployment Configuration:**
- `Procfile` at repo root: `web: python main.py`
- Environment variables set in Railway dashboard
- Graceful shutdown on SIGTERM (drain timeout: `DRAIN_TIMEOUT` seconds, default 300)
- `VOICE_AGENT_ROOT` env var: override project root path discovery for packaged deployments

## Webhooks & Callbacks

**Incoming (Twilio → shuo):**
- `POST /twiml` — Twilio calls this when number is dialed
  - Validated with Twilio signature
  - Returns TwiML with WebSocket connection URL

**DTMF Redirect:**
- `POST /twiml/ivr-dtmf?digit=N` — DTMF playback and reconnect
  - Used when agent calls `press_dtmf(digit)` tool
  - Redirects call to next TwiML, reconnects stream

**Monitor Events:**
- `WS /dashboard/ws` — Live event stream (transcript, phase, control)
  - All call events broadcast to connected supervisor browsers
  - Backed by in-memory event bus in `monitor/bus.py`

**Takeover Conference:**
- `POST /twiml/conference/{call_id}` — Creates 3-way conference for supervisor
  - Agent silenced during takeover
  - Supervisor's speech transcribed via `/ws-listen` stream
