# External Integrations

**Analysis Date:** 2026-03-18

## APIs & External Services

**Voice & Telephony:**
- Twilio - Voice API for PSTN and WebRTC calls
  - SDK: `twilio` 9.0.0+
  - Auth: `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`
  - Configuration: `TWILIO_PHONE_NUMBER` (E.164 format)
  - Key files: `shuo/services/twilio_client.py`
  - Features: Outbound calls, WebSocket media streams, DTMF tones, call recording
  - Public URL: `TWILIO_PUBLIC_URL` (ngrok for local dev, production URL in deployment)

**Speech-to-Text:**
- Deepgram Flux API - Real-time streaming STT + turn detection
  - SDK: `deepgram-sdk` 3.0.0+
  - Auth: `DEEPGRAM_API_KEY`
  - Key files: `shuo/services/flux.py`, `shuo/services/flux_pool.py`
  - Connection: Persistent WebSocket per call
  - Audio format: μ-law 8kHz (direct from Twilio)
  - Turn detection: Automatic StartOfTurn/EndOfTurn events

**Large Language Model:**
- Groq API (OpenAI-compatible) - LLM inference
  - Client: OpenAI SDK (`openai` 1.0.0+)
  - Auth: `GROQ_API_KEY`
  - Default model: `LLM_MODEL=llama-3.3-70b-versatile` (configurable)
  - Key files: `shuo/services/llm.py`
  - Features: Streaming token response, conversation history management
  - Markers: LLM can emit `[DTMF:N]`, `[HANGUP]`, `[HOLD]` control markers

**Text-to-Speech - Cloud:**
- ElevenLabs - Premium cloud TTS
  - Auth: `ELEVENLABS_API_KEY`
  - Voice ID: `ELEVENLABS_VOICE_ID` (default: Rachel `21m00Tcm4TlvDq8ikWAM`)
  - Model: `ELEVENLABS_MODEL=eleven_flash_v2_5` (configurable)
  - Key files: `shuo/services/tts_elevenlabs.py`
  - Protocol: WebSocket streaming to Twilio-compatible μ-law 8kHz

**Text-to-Speech - Local (Optional):**
- Kokoro-82M - Local TTS (zero API cost)
  - Deployment: Docker container
  - Key files: `shuo/services/tts_kokoro.py`
  - Environment: `TTS_PROVIDER=kokoro`
  - Pre-loaded on startup for reduced latency

**Text-to-Speech - Self-hosted (Optional):**
- Fish Audio S2 - Self-hosted TTS option
  - Key files: `shuo/services/tts_fish.py`
  - Environment: Fish Audio server URL in configuration
  - Use case: Full privacy, self-hosted alternative to ElevenLabs

## Data Storage

**Databases:**
- None - Voice agent is stateless
- Call state stored in-memory during active calls
- No persistent database layer

**File Storage:**
- Local filesystem only
- Call traces saved to `/tmp/shuo/{stream_sid}.json` for latency analysis
- Twilio call recordings stored in Twilio account (not downloaded)

**Caching:**
- In-memory service connection pools:
  - TTS pool: `shuo/services/tts_pool.py` (pre-warmed connections, TTL=120s)
  - Flux pool: Not pooled (fresh connection per call for accurate turn detection)
  - Pre-loaded Kokoro model in GPU memory on startup

**In-Memory Event Bus:**
- `dashboard/bus.py` - Call-scoped event bus for live transcription/phase updates
- `dashboard/registry.py` - In-memory call registry (active calls)
- Queues: asyncio.Queue with 256-512 item max for event backpressure

## Authentication & Identity

**Auth Provider:**
- Custom - No central auth service
- Twilio JWT Access Token for browser softphone
  - Generated by `shuo/server.py:get_token()` endpoint
  - Signed with Twilio API Key + Secret
  - Required for browser WebRTC connection

**Token Management:**
- Tokens generated per browser session
- No user authentication — designed for supervised calls only
- Dashboard access is unsecured (suitable for local/internal use only)

## Monitoring & Observability

**Error Tracking:**
- None - No external error tracking service configured

**Logs:**
- Custom colored terminal logging via `shuo/log.py`
- Structured logging with ServiceLogger instances for each module
- No external log aggregation (logs to stdout)

**Performance Tracing:**
- Per-call latency tracing: `shuo/tracer.py`
- Traces saved to `/tmp/shuo/{stream_sid}.json`
- Spans: LLM first token, TTS first audio, playback complete
- Accessible via `GET /trace/latest` endpoint

**Debugging:**
- Call state machine in `shuo/state.py` (pure, no I/O)
- Conversation event loop in `shuo/conversation.py`
- Agent pipeline in `shuo/agent.py`

## Webhooks & Callbacks

**Incoming (Twilio → shuo):**
- `POST /twiml` - Twilio calls this when number is dialed
  - Returns TwiML with WebSocket connection URL
  - Initiates media stream to `/ws` endpoint
  - Parsed by `shuo/services/twilio_client.py:parse_twilio_message()`

**IVR Webhooks:**
- `POST /twiml/ivr-dtmf?digit=N` - DTMF playback and reconnect
  - Used when agent sends `[DTMF:N]` marker
  - Redirects call to next TwiML, reconnects stream

**Dashboard Events:**
- `WS /dashboard/ws` - Live event stream (transcript, phase, control)
  - Publishes all call events to connected supervisor browsers
  - Events: `agent_speaking`, `user_speaking`, `call_started`, `call_ended`, etc.
  - Backed by in-memory event bus in `dashboard/bus.py`

**Takeover Conference:**
- `POST /twiml/conference/{call_id}` - Creates 3-way conference for supervisor
  - Supervisor registered as WebRTC client
  - Agent silenced during takeover
  - Supervisor's speech transcribed via `/ws-listen` stream

## Call Control

**Rest API Endpoints:**
- `GET /call/{number}` - Trigger outbound call
- `POST /dashboard/call` - Place call with goal (JSON body: phone, goal, ivr_mode)
- `POST /dashboard/calls/{id}/hangup` - Terminate call
- `POST /dashboard/calls/{id}/takeover` - Suppress agent, enable supervisor
- `POST /dashboard/calls/{id}/handback` - Return control to agent
- `POST /dashboard/calls/{id}/dtmf` - Inject DTMF digit (JSON body: digit)
