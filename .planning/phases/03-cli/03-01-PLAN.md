Create `shuo/shuo/cli.py` with:

1. **`_load_config(config_path: str | None) -> dict` function:**
   - If `config_path` is provided, load it with `yaml.safe_load()`. If file not found, print error and `sys.exit(1)`.
   - If `config_path` is None, check for `voice-agent.yaml` in `os.getcwd()`. If exists, load it. If not, return `{}`.
   - Wrap `yaml.safe_load()` in try/except `yaml.YAMLError` — print error and exit on bad YAML.

2. **`@click.group()` named `cli`:**
   - Option `--config` / `-c`, type `click.Path(exists=False)`, default `None`, help "Path to YAML config file"
   - In callback: call `load_dotenv()`, call `setup_logging()`, store `_load_config(config)` in `ctx.obj["config"]`
   - Call `ctx.ensure_object(dict)` before accessing `ctx.obj`

3. **`@cli.command()` named `serve`:**
   - Options: `--port` (int, default None), `--drain-timeout` (int, default None)
   - Config merge: `cfg = ctx.obj["config"].get("serve", {})`
   - Effective port: `port if port is not None else cfg.get("port", int(os.getenv("PORT", "3040")))`
   - Effective drain: `drain_timeout if drain_timeout is not None else cfg.get("drain_timeout", int(os.getenv("DRAIN_TIMEOUT", "300")))`
   - Per-subcommand env check: validate TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, TWILIO_PHONE_NUMBER, TWILIO_PUBLIC_URL, DEEPGRAM_API_KEY, GROQ_API_KEY, ELEVENLABS_API_KEY are set. Print missing and `sys.exit(1)` if any missing.
   - Register SIGTERM handler (copy `_handle_sigterm` logic from main.py lines 110-143, using `import shuo.server as server_module` for `_active_calls` and `_draining`)
   - Start server: `Logger.server_starting(effective_port)`, start `start_server(effective_port)` in a daemon thread, `time.sleep(2)`, `Logger.server_ready(os.getenv("TWILIO_PUBLIC_URL", ""))`, then `while True: time.sleep(1)` in try/except KeyboardInterrupt calling `Logger.shutdown()`.
   - Import `start_server` from `main` module: `from shuo.main_helpers import start_server` — BUT since main.py has side effects at import time (load_dotenv, setup_logging at module level), create a thin wrapper. Actually, just import directly: `import uvicorn` and `from shuo.server import app` and inline the `start_server` logic (3 lines: Config + Server + run). This avoids importing main.py which has module-level side effects.

4. **`@cli.command()` named `call`:**
   - Argument: `phone` (required, type str)
   - Options: `--goal` (str, default None), `--identity` (str, default None)
   - Config merge: `cfg = ctx.obj["config"].get("call", {})`
   - Effective phone: use `phone` argument (always provided — it is a required argument)
   - Effective goal: `goal if goal is not None else cfg.get("goal", os.getenv("CALL_GOAL", ""))`
   - Effective identity: `identity if identity is not None else cfg.get("identity", "")`
   - Identity handling: identity is NOT a native run_conversation param. If identity is non-empty, prepend it to goal: `effective_goal = f"You are {effective_identity}. {effective_goal}" if effective_identity else effective_goal`. Set `os.environ["CALL_GOAL"] = effective_goal` so the server picks it up when processing the call.
   - Per-subcommand env check: same 7 vars as serve. Print missing and exit if any.
   - Start server in daemon thread (same as serve), wait 2s, then call `make_outbound_call(phone)`, log call SID, then block with `while True: time.sleep(1)` until KeyboardInterrupt.

5. **`@cli.command()` named `bench`:**
   - Option: `--dataset` (str, default None), help "YAML scenario file"
   - Config merge: `cfg = ctx.obj["config"].get("bench", {})`
   - Effective dataset: `dataset if dataset is not None else cfg.get("dataset")`
   - Print: `click.echo(f"bench: dataset={effective_dataset!r}")` then `click.echo("Benchmark runner not yet implemented (Phase 4).")`

Create `shuo/tests/test_cli.py` using `click.testing.CliRunner` and `unittest.mock.patch`.

All tests that invoke `serve` or `call` must mock:
- `threading.Thread` (to prevent actual server start) — or mock `uvicorn.Server` and `uvicorn.Config`
- `time.sleep` (to prevent blocking)
- `make_outbound_call` (for call tests)
- Environment variables via `monkeypatch.setenv` or `@patch.dict(os.environ, ...)`

For serve tests, since serve blocks on `while True: time.sleep(1)`, the mock of `time.sleep` should raise `KeyboardInterrupt` on second call to break the loop.

For config tests, use `CliRunner(mix_stderr=False)` and `runner.isolated_filesystem()` to create temporary `voice-agent.yaml` files.

For env var check tests on serve, clear all TWILIO_* env vars and verify the command exits with error.

Example test structure:
```python
import os
from unittest.mock import patch, MagicMock
from click.testing import CliRunner
from shuo.cli import cli

def test_bench_stub():
    runner = CliRunner()
    result = runner.invoke(cli, ["bench", "--dataset", "test.yaml"])
    assert result.exit_code == 0
    assert "not yet implemented" in result.output.lower()
    assert "test.yaml" in result.output

def test_config_flag_override():
    runner = CliRunner()
    with runner.isolated_filesystem():
        with open("voice-agent.yaml", "w") as f:
            f.write("bench:\n  dataset: from_config.yaml\n")
        result = runner.invoke(cli, ["bench", "--dataset", "from_flag.yaml"])
        assert "from_flag.yaml" in result.output
        assert "from_config.yaml" not in result.output
```
