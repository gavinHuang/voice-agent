2. Replace lines 332-333:
   ```python
   if self._on_token_observed:
       self._on_token_observed(clean_text)
   ```
   with:
   ```python
   if self._on_token_observed:
       asyncio.get_event_loop().call_soon(self._on_token_observed, clean_text)
   ```

That's the entire fix. `call_soon` schedules the synchronous callback to run after the current coroutine yields, so `_on_llm_token` returns immediately without waiting for the observer. The observer still runs on the same event loop (no thread), but it runs in a separate turn — the LLM stream is never blocked.

Add a code comment above the line:
```python
# Schedule observer on next event-loop turn — never block the LLM token stream (BUG-03)
```

2. Add the watchdog function before `run_conversation` (above line 42):
   ```python
   async def _inactivity_watchdog(
       event_queue: asyncio.Queue,
       timeout: float,
       last_activity: list,  # mutable single-element list [float] for shared state
   ) -> None:
       """Hang up if no meaningful call activity for `timeout` seconds.

       Activity events (StreamStart, FluxEndOfTurn, AgentTurnDone, HoldStart, HoldEnd)
       update last_activity[0]. MediaEvents do NOT count — a silent-but-connected call
       streaming audio silence should still be caught.
       """
       try:
           while True:
               await asyncio.sleep(5.0)
               now = asyncio.get_event_loop().time()
               if now - last_activity[0] > timeout:
                   logger.warning(f"Inactivity timeout ({timeout}s) — requesting hangup")
                   await event_queue.put(HangupRequestEvent())
                   return
       except asyncio.CancelledError:
           pass
   ```
